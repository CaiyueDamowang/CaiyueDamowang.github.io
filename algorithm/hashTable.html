散列函数就是根据不同的算法将一个key映射成数组的地址

不同的key生成的hash值相同的情况，称为Hash碰撞

解决Hash碰撞的两种方法：

# 开链法

在散列表的底层数组中，每个数组元素又是一个新的数据结构，比如另一个数组

如此，即使两个键的Hash值相同，也依然被存在同样的位置。不过存储在第二个数组的不同位置中。

BuildingChains() {
    for(let i = 0; i < this.table.length; i ++) {
        this.table[i] = new Array()
    }
}

put(key, data) {
    const pos = this._hash(key)
    let index = 0
    while(this.table[pos][index] !== undefined) {
        index += 2
    }
    this.table[pos][index] = key
    this.table[pos][index + 1] = data
}
get(key) {
    const pos = this._hash(key)
    let length = this.table[pos].length
    let index = 0

    while(this.table[pos][index] !== key && index < length) {
        index += 2
    }
    return this.table[pos][index]
}

# 线性探测法

线性探测法是指在原hash值之上,在数组中找到一个新值（线性增加）存储数据。要求原有数组足够大

this.tables = []
this.values = []

put(key, data) {
    let pos = this._hash(key)
    if(this.tables[pos] !== undefined) {
        while(this.tables[pos]) {
            pos++
        }
        this.tables[pos] = key
        this.values[pos] = data
    }
}
get(key) {
    let pos = this._hash(key)
    for(let i = pos; this.table[pos] != undefined; i++) {
        return this.values[pos]
    }
    return undefined
}

当存储数据的数组空间特别大的时候，线性探测法要比开链法好，如果数组的大小是存储数据个数的1.5倍，那么可以使用开链法。如果是两倍以上，则使用线性探测法。
