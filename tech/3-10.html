each box will be referred to as a "phase" of the event loop.

# Phases Overview

  • timers: this phase executes callbacks scheduled by setTimeout() and setInterval().

  • pending callbacks: executes I/O callbacks deferred to the next loop iteration.

  • idle, prepare: only used internally.

  • poll: retrieve new I/O events; execute I/O related callbacks (almost all with the - exception of close callbacks, the ones scheduled by timers, and setImmediate()); node will block here when appropriate.

  • check: setImmediate() callbacks are invoked here.

  • close callbacks: some close callbacks, e.g. socket.on('close', ...).

# Phase in Detail

  • timers

A timer specifies the threshold after which a provided callback may be executed rather than the exact time a person wants it to be executed

计时器提供了回调函数执行时间的阈值，而不是被执行的确切时间

Operating System scheduling or the running of other callbacks may delay them.

操作系统的调度或者其他函数会导致延迟

这里有一条 tips：实际上，timers 的执行时间由 poll 控制，原因往下看

  • pending callbacks

This phase executes callbacks for some system operations such as types of TCP errors. For example if a TCP socket receives ECONNREFUSED when attempting to connect, some *nix systems want to wait to report the error. This will be queued to execute in the pending callbacks phase.

这个阶段调用一些操作系统的操作，例如TCP错误。

# - poll

The poll phase has two main functions:

# 1.Calculating how long it should block and poll for I/O, then
# 2.Processing events in the poll queue. 处理轮询队列的事件:

When the event loop enters the poll phase and there are no timers scheduled, one of two things will happen:

# 1.If the poll queue is not empty, the event loop will iterate through its queue of callbacks executing them synchronously until either the queue has been exhausted, or the system-dependent hard limit is reached.

如果轮询队列非空，事件循环会遍历回调函数队列，直到队列为空，或调用次数超过限制

# 2.If the poll queue is empty, one of two more things will happen:

如果轮询队列为空

Once the poll queue is empty the event loop will check for timers whose time thresholds have been reached. If one or more timers are ready, the event loop will wrap back to the timers phase to execute those timers' callbacks.

# ① If scripts have been scheduled by setImmediate(), the event loop will end the poll phase and continue to the check phase to execute those scheduled scripts.

如果setImmediate调度了脚本。则结束poll阶段，进入到check阶段 并且执行这些脚本

# ② If scripts have not been scheduled by setImmediate(), the event loop will wait for callbacks to be added to the queue, then execute them immediately.

如果setImmediate没有被调度的脚本。则会等到回调函数被添加的时候理解执行

Once the poll queue is empty the event loop will check for timers whose time thresholds have been reached. If one or more timers are ready, the event loop will wrap back to the timers phase to execute those timers' callbacks.

一旦轮询队列为空，就会检查定时器的阈值是否达到（一旦清空微队列，则取出一个 timer 队列中的宏任务），返回到 timers 阶段，并且触发这些定时回调函数

  • check

This phase allows a person to execute callbacks immediately after the poll phase has completed. If the poll phase becomes idle and scripts have been queued with setImmediate(), the event loop may continue to the check phase rather than waiting.

此阶段允许人员在轮询阶段完成后立即执行回调。如果轮询阶段变得空闲，并且脚本已使用setImmediate（）排队，则事件循环可能会继续进行检查阶段，而不是等待。

Generally, as the code is executed, the event loop will eventually hit the poll phase where it will wait for an incoming connection, request, etc. However, if a callback has been scheduled with setImmediate() and the poll phase becomes idle, it will end and continue to the check phase rather than waiting for poll events.

通常，在代码执行阶段。事件循环最终会进行到轮询,这时轮询会不断等待传入的链接和请求，如果setImmediate调度了回调函数，那么会结束轮询。进入check阶段

  • close callbacks

If a socket or handle is closed abruptly (e.g. socket.destroy()), the 'close' event will be emitted in this phase. Otherwise it will be emitted via process.nextTick().

# setimmediate vs setTimeout

We have two calls that are similar as far as users are concerned, but their names are confusing.

process.nextTick() fires immediately on the same phase setImmediate() fires on the following iteration or 'tick' of the event loop In essence, the names should be swapped. process.nextTick() fires more immediately than setImmediate(), but this is an artifact of the past which is unlikely to change. Making this switch would break a large percentage of the packages on npm. Every day more new modules are being added, which means every day we wait, more potential breakages occur. While they are confusing, the names themselves won't change.

We recommend developers use setImmediate() in all cases because it's easier to reason about.

简单来说

他俩的名字应该互调

因为nextTick是在本阶段立即调用

setImmdiate实在下一个阶段调用

# 总结

poll phase 轮询阶段 即所谓的微任务队列

其中的setImmdiate和nextTick

  • nextTick: 在本阶段执行 下面有测试

  • setImmdiate:在 poll 结束的 check 阶段执行

每当 poll 阶段清空了队列，就会检查是否有定时器达到了阈值，如果达到了阈值，则返回 timer 阶段，并且调用这些定时回调函数
