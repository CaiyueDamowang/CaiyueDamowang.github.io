# arguments

在ES5非严格模式中， 对arguments的元素更改，会同步在arguments的数组中

在ES6中和严格模式中相同，更改arguments的元素，不会更改arguments

# 箭头函数

这些属性按作用域规则访问外层变量

# 箭头函数规则:

  • 箭头函数没有 this, super, arguments, new.target

  • 不能通过new调用

  • 没有原型

  • 不能改变this的绑定

  • 不支持重复的命名参数

# ES6的尾调用优化

并不是尾递归优化， 尾递归优化需要自己实现

将n个函数上下文改成一个函数上下文的方法

  • 尾调用是指：

函数作为另一个函数的返回值被调用

在ES5中，会在调用栈中推入一个新的栈帧（stack frame），每个没用完的栈帧都会保存到内存中

从而出现栈溢出的问题

在ES6中，如果满足以下条件，尾调用不再创建新的栈帧，而是会清楚并且重用当前栈帧

  • 尾调用不访问当前栈帧的变量（不生成闭包）

  • 在函数内部，尾调用是最后一条语句

  • 尾调用的结果作为函数值返回

function doSmth(){
    return doSmth()
}
