# 1.网络的组成

# 因特网的边缘部分

处在因特网边缘的部分就是连接在因特网上的所有的主机，又称为”端系统“

主机A和主机B通信，实际上是指运行在主机A的某个程序和运行在主机B上的另一个程序进行通信

# 客户/服务器方式

A是客户，B是服务器，客户A向B发出请求服务，服务器B想客户A提供服务

客户程序是指被用户调用后运行，在通信是主动地向远程服务器发起通信（请求服务），因此，客户必须知道服务器程序的地址。不需要特殊的硬件和复杂的操作系统

服务器程序是一种专门用来提供服务的程序，可同事处理多个远程或者本地客户的请求。启动后即自动调用并一直不断地运行着，被动的等待并接受来着多方的客户的通信请求，服务器程序不需要知道客户程序的地址。 一般需要有强大的硬件和高级的操作系统支持。

# 对等连接方式（peer-to-peer/ P2P）

是指两个主机在通信时，不缺分服务器和客户的概念，只要两个主机都运行了P2P软件，就可以进行对等的连接通信。双方都可以下载对方已经存储在硬盘中的共享文档。因此工作方式也成P2P文件共享。

# 因特网的核心部分

网络核心部分是因特网中最复杂的部分，因为网络中的核心部分要向网络边缘中的大量主机提供连通性，使边缘部分的任一主机都能向其他主机进行通信。

在网络核心中起特殊作用的是路由器(router)，他是一种专用计算机（但不是主机）。路由器是实现分组交换的关键构件，其任务是转发收到的分组

# 电路交换概念

如果要所有的电话机都两两相连接就需要极大量的电话线， 所以，为了节约资源，将所有的电话机连接到交换机上，交换机使用交换dev方法让用户彼此可以方便的通信。

这样的方法构成了覆盖全时间的电信网

交换就是按照某种方式动态的分配传输线路的资源，在使用电路交换打电话之前，必须先拨号请求建立连接。

从主叫端到被叫端建立了一条链接，也就是一条专用的物理通道。这条连接保证了双方童话时所需要的通信资源，这些资源不会被其他用户占用。

建立连接（占用通信资源）-> 通话 -> 释放连接 -(归还资源)

交换机相当于网络中的结点，交换机之间由大量的中继线相连接。

用户线是每个用户独占的传送模拟信号的专用线路，正在通话的用户只占用了中继线里面的一个话路。

# 分组交换概念

分组交换采用存储转发技术。

在发送报文之前，先把较长的报文划分成一个个更小的等长数据段 每个数据段前面加上一些必要的控制信息组成的首部(header). 这样就构成了一个分组。

路由器是用来转发分组的，即进行分组交换。

分组到达一个路由器后，先暂时存储下来，查找转发表，然后从另一条合适的链路转发出去。分组在传输时就这样一段段的断续占用通信资源。

灵活高效： 分组交换在传送数据前不比先占用一条端到端的通信资源

迅速可靠：以分组为传送单位，不先简历连接就能向其他主机发送分组，并且可靠的网络协议；分布式的多路由的分组交换网，网络有很好的生存性

# 报文交换

整个报文先传送到相邻结点，全部存储下来后查找转发表，转发到下一个节点

# 2.性能指标

  • 速率： 比特（bit)是数据量的单位，意思是一个二进制数据

  • 带宽

    1.信号的带宽是指该信号所包含的不同频率成分所占据的频率范围。

    2.网络中表示两节点之间所能通过的最高数据量

    3.吞吐量：网络中通过的实际数据量

    4.时延：

      发送时延： 主机或者路由器将数据帧发送所需要的时间（从第一个bit算起，一直到数据的最后一个比特）

  传播时延： 信道中传播一定的距离需要的时间

  处理时延、排队时延（路由器要先在输入队列中排队等待处理）

  • 性能概念：

带宽、速率、时延

  • 通信概念：

    1.单工通信： 只能有一个方向的通信而没有反方向

    2.单双工： 不能同时发送和接受

    3.全双工： 可以同时发送和接受

  • 传输媒体:

    1 双绞线

    2.同轴电缆

    3.光纤

  • 信道复用技术

1.频分复用： 用户在分配到一定的频带后，在通信过程中自始至终都占用这个频带。是用户在同样的时间占用不同的带宽（频率带宽而不是数据发送率）资源

2.时分复用： 用户在不同的时间占用同样的频带宽度，更有利于数字信号的传输，但用户越多，每个用户的时隙宽度越窄

3.波分复用： 光的频分复用。一根光纤传输多个频率很接近的光载波信号，由于光载波的频率很高所以习惯上成波分复用而不是光的频分复用

4.码分复用： 每一个哟洪湖可以再同样的时间使用同样的频率进行通信。

# 2.链路层

此链路为数据链路(通信协议控制数据的传输)

并非为物理链路(相邻节点之间的物理线路)

链路层的主要信道 ： 1.点对点信道 2.广播信道

链路层解决的三个问题

1.封装成帧

在一段数据的前后分别添加首部和尾部

接收端收到物理层上交的比特流后，就能根据首尾部的标记，从收到的比特流中识别帧的开始和结束

网络层的IP数据报(分组)传送到数据链路层就成为帧的数据部分


|帧首部|·······数据部分·······|帧尾部|

|---------------帧长--------------|

2.透明传输

透明传输： 表示无论什么样的比特组合的数据都能够通过这个数据链路层

为了使非ASCII编码的数据不会被错误的认为是SOH(帧开始符)或EOT(帧结束符),要在前面加转义字符ESC

3.差错控制

通信链路都不会是理想的，比特在信道传输过程中可能会产生差错，比如1变成0等称为比特差错

帧的时序错乱或者丢帧等称为传输差错

CRC（循环冗余检验）是检测比特差错的一种方法

在发送端，将数据划分为组，假定每组K个比特，在后面加上N个冗余码，然后构成一个帧发送出去

在接收端，用事先商定的长度为N+1的除数P得出商是Q而余数是R，
1.如果R为0，则认定帧没有差错（R=0并且帧出错的概率几乎为0）
2.如果R!=0，则判定帧由差错，丢弃此帧

  • 使用广播信道的数据链路层

1.以太网:

最初的以太网是将许多计算机连接到一根总线上，当初认为这样的链接方法既简单又可靠，因为总线上没有有源器件

总线上的每一个工作的计算机都能检测到一台计算机B发送的数据信号

由于只有某个计算机D的地址与数据帧首部写入的地址一致，因此只有D才能接受这个数据帧

其他所有计算机都检测到不是发送给他们的数据帧，所以在具有广播性的线路上实现了一对一通信

(1) 多点接入： 表示许多计算机以多点接入的方式连接在一根总线上
(2) 载波监听： 是指每一个站在发送数据之前先要检测一下总线上是否有其他计算机在发送数据，如果有，则暂时不要发送数据，以免发生碰撞

2.适配器:

计算机与局域网的链接是通过适配器。适配器是主机箱的一块网络接口板。上面装有处理器和存储器，

注意:适配器和局域网之间的通信是通过电缆或双绞线以串行传输方式进行的。

而适配器与计算机主板上的IO总线是并行传输方式进行的

因两者传输数据率并不相同，因此在适配器中必须装有对数据进行缓存的存储芯片

适配器做了哪些？

接受和发送各种帧的时候不适用计算机的CPU，收到差错帧，就丢弃而不必通知计算机。

当计算机要发送IP数据报的时候，协议栈就将数据报交给适配器，组装成帧后发送到局域网

计算机的硬件地址就在适配器的ROM种，软件地址--IP地址在计算机的存储器中
